# 申请状态流转

本文件定义资金申请从创建到审批结束的状态变化规则。

## 1. 状态定义

| 状态 | 中文名 | 谁可见 | 说明 |
| --- | --- | --- | --- |
| pending | 待审批 | 申请方、审批方 | 已提交，等待审批 |
| approved | 已通过 | 申请方、审批方 | 审批方同意使用共同资金 |
| rejected | 已驳回 | 申请方、审批方 | 审批方不同意本次申请 |
| withdrawn | 已撤回 | 申请方、审批方 | 申请方在审批前撤回 |

MVP 必须实现 `pending`、`approved`、`rejected`。`withdrawn` 可以在第二版实现。

## 2. 状态流转图

```text
创建申请
   |
   v
pending 待审批
   |\
   | \_ rejected 已驳回
   |
   \___ approved 已通过

pending 待审批
   |
   \___ withdrawn 已撤回
```

## 3. 流转规则

### 3.1 创建申请

- 触发方：申请方。
- 初始状态：`pending`。
- 必须记录：申请人、家庭、标题、金额、必要性说明、创建时间。
- 可选记录：分类、照片、产品链接、期望购买时间、备注。

### 3.2 审批通过

- 触发方：审批方。
- 前置状态：`pending`。
- 目标状态：`approved`。
- 必须记录：审批人、审批意见、审批时间。

### 3.3 审批驳回

- 触发方：审批方。
- 前置状态：`pending`。
- 目标状态：`rejected`。
- 必须记录：审批人、驳回原因、审批时间。

### 3.4 撤回申请

- 触发方：申请方。
- 前置状态：`pending`。
- 目标状态：`withdrawn`。
- 必须记录：撤回人、撤回时间。
- MVP 可暂不实现。

## 4. 禁止流转

| 当前状态 | 禁止操作 | 原因 |
| --- | --- | --- |
| approved | 再次审批 | 避免审批结果被无记录覆盖 |
| rejected | 再次审批 | 驳回后应重新提交新申请 |
| withdrawn | 审批 | 已撤回申请不应继续处理 |
| pending | 修改金额 | 提交后修改金额会影响审批可信度 |

如果后续要支持修改申请，建议使用“撤回后重新提交”而不是直接编辑已提交申请。

## 5. 事件记录

每一次关键动作都应写入 `applicationEvents`。

```js
{
  applicationId: 'app_001',
  actorOpenId: 'openid_xxx',
  actorName: '审批方',
  action: 'approved',
  fromStatus: 'pending',
  toStatus: 'approved',
  comment: '可以买，控制在预算内。',
  createdAt: '2026-06-22 10:30'
}
```

## 6. 页面同步规则

- 首页的待审批数量来自 `pending` 申请数量。
- 列表状态标签来自申请 `status`。
- 详情页按钮根据 `status` 判断是否展示。
- 只有 `pending` 状态展示通过和驳回按钮。
- 状态改变后，应返回列表并刷新，或留在详情页并更新当前详情数据。
