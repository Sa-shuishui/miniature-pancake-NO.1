# 开发路线图

## 阶段 0：需求与方向确认

目标：确认小程序要解决的问题、第一版功能范围、视觉方向。

已完成：

- 明确夫妻家庭共同资金审批场景。
- 明确申请字段：用途、金额、必要性说明、产品照片、产品链接。
- 明确审批状态：待审批、已通过、已驳回。
- 明确第一版使用微信小程序原生框架与本地模拟数据。

待确认：

- 视觉方向：温馨家庭感、专业财务感、极简微信原生感。

## 阶段 1：原生小程序 MVP

目标：在微信开发者工具中跑通完整本地交互。

任务：

- 创建小程序项目配置。
- 创建底部 Tab：首页、申请、列表、我的。
- 实现本地 mock store。
- 实现首页审批概览。
- 实现申请表单。
- 实现申请列表与状态筛选。
- 实现申请详情。
- 实现通过和驳回操作。
- 实现表单校验与轻提示。

完成标准：

- 不接云开发也可以完整演示一次申请和审批流程。
- 申请状态改变后，首页、列表、详情页展示一致。

## 阶段 2：微信云开发接入

目标：把本地 mock 数据替换为真实云端数据。

任务：

- 开通云开发环境。
- 创建 `families`、`familyMembers`、`fundApplications`、`applicationEvents` 集合。
- 接入微信登录并获取 openid。
- 实现创建申请云函数。
- 实现查询申请云函数。
- 实现审批云函数。
- 接入云存储上传申请图片。

完成标准：

- 两个微信用户可以在同一个家庭下提交和审批申请。
- 图片可以上传到云存储并在详情页展示。
- 审批记录可追踪。

## 阶段 3：家庭使用增强

目标：让工具更适合真实家庭长期使用。

可选功能：

- 微信订阅消息提醒审批方。
- 月度预算额度。
- 支出分类统计。
- 付款凭证上传。
- 驳回后重新提交。
- 申请撤回。
- 已通过申请标记为已购买。

## 阶段 4：发布与维护

目标：完成微信小程序发布前检查。

任务：

- 补充小程序名称、头像、类目、隐私说明。
- 检查用户数据收集范围。
- 检查云数据库权限。
- 完成真机测试。
- 提交微信审核。

完成标准：

- 小程序可由家庭成员稳定使用。
- 数据权限不泄露到其他家庭。
- 核心审批流程没有阻断问题。
