# 测试计划

## 1. 测试目标

验证家庭内部财政审批小程序的 MVP 流程是否可用：

- 申请方能提交共同资金申请。
- 审批方能查看并处理申请。
- 申请字段、状态和审批意见能被正确保存与展示。

## 2. 测试角色

### 申请方

模拟不直接管理家庭共同资金的一方。

需要验证：

- 能创建申请。
- 能查看自己提交的申请状态。
- 能看到审批意见。

### 审批方

模拟管理家庭共同资金的一方。

需要验证：

- 能看到待审批申请。
- 能通过申请。
- 能驳回申请。
- 能填写审批意见。

## 3. MVP 手工测试用例

### TC-000 首次设置性别和角色

前置条件：首次打开小程序。

步骤：

1. 在性别选择页选择男生。
2. 进入淡蓝主题角色选择页。
3. 选择申请者。
4. 回到性别选择页或重新打开流程，选择女生。
5. 进入淡粉主题角色选择页。
6. 选择审批者。

预期结果：

- 男生选择后只展示男生主题下的申请者和审批者角色。
- 女生选择后只展示女生主题下的申请者和审批者角色。
- 选完角色后进入共同花主界面。
- 主界面主题色与已选择性别一致。

### TC-001 创建咖啡机购买申请

前置条件：进入“新建申请”页面。

步骤：

1. 填写用途标题：购买咖啡机。
2. 填写金额：1299。
3. 填写分类：家电。
4. 填写必要性说明：家里每天喝咖啡，长期看比外卖咖啡更省钱。
5. 填写产品链接。
6. 选择一张产品照片。
7. 点击提交。

预期结果：

- 提交成功。
- 申请状态为待审批。
- 列表中出现“购买咖啡机”。
- 详情页展示金额、说明、图片和链接。

### TC-002 必填项校验

前置条件：进入“新建申请”页面。

步骤：

1. 不填写用途标题，点击提交。
2. 填写用途标题但不填写金额，点击提交。
3. 填写金额但不填写必要性说明，点击提交。

预期结果：

- 每一步都出现对应错误提示。
- 不会创建无效申请。

### TC-003 链接格式校验

前置条件：进入“新建申请”页面。

步骤：

1. 填写完整申请内容。
2. 产品链接填写 `abc123`。
3. 点击提交。

预期结果：

- 系统提示产品链接需要以 `http` 或 `https` 开头。
- 申请不会提交。

### TC-004 审批通过

前置条件：存在一条待审批申请。

步骤：

1. 审批方进入申请详情页。
2. 填写审批意见：可以买，控制在预算内。
3. 点击通过。

预期结果：

- 申请状态变为已通过。
- 详情页显示审批意见。
- 详情页显示审批时间。
- 列表中的状态同步更新。

### TC-005 审批驳回

前置条件：存在一条待审批申请。

步骤：

1. 审批方进入申请详情页。
2. 填写审批意见：这个月预算紧张，下个月再考虑。
3. 点击驳回。

预期结果：

- 申请状态变为已驳回。
- 详情页显示驳回原因。
- 列表中的状态同步更新。

### TC-006 状态筛选

前置条件：存在待审批、已通过、已驳回三种状态的申请。

步骤：

1. 打开申请列表。
2. 切换到待审批。
3. 切换到已通过。
4. 切换到已驳回。

预期结果：

- 每个筛选项只展示对应状态的申请。
- 全部列表展示所有申请。

## 4. 真机测试重点

- 表单输入在不同屏幕尺寸下不被遮挡。
- 图片选择和预览可以正常工作。
- 长产品链接不会撑破布局。
- 金额较大时展示仍然清晰。
- 审批按钮不会被底部安全区遮挡。

## 5. 云开发测试重点

接入云开发后，需要追加验证：

- 不同家庭之间数据隔离。
- 非家庭成员无法读取申请。
- 申请方不能审批自己的申请，除非家庭规则允许。
- 图片上传后能稳定读取。
- 云函数错误时页面有可理解的提示。
